#### Texture-Chunker Pipeline Detail

```
exports/*.md
    |
    v  [chunk_conversation_md.py]
chunks/ (JSONL, multiple sizes: 1,2,3,5 pairs)
    |
    v  [clean_chunks.py]                        <- local, deterministic
chunks_clean/ (stripped <thinking>, attachments)
    |
    v  [classify_chunks_openrouter_async.py]    <- EXTERNAL LLM (OpenRouter)
chunks_tagged/ (tagged via LLM)
    |
    v  [feels_scorer.py]                        <- local, deterministic
    |
    v  [add_salience_to_chunks.py]              <- local, incremental (skips scored)
chunks_scored/ (feels_score + salience_score)
    |
    v  [shard_sampler.py]                       <- RUNTIME (called per heartbeat pulse)
pulse_injection.txt (ready for heartbeat)
```

*Pipeline is incremental: `run_texture_pipeline.py` tracks processed files in `pipeline_state.json`*

#### Interoception Pipeline Detail (PER-41/42/57/58/59)

```
pulse output (raw text from heartbeat)
    |
    v  [analyzer.py: process_pulse_output]
signals dict (13 signals):
  Core (4):
  - meta_commentary    (self-referential density)
  - hedging_ratio      (tentative vs assertive)
  - self_correction    (mid-output revisions)
  - question_density   (questions per sentence)
  Affect (2):
  - affect_valence     (NRC VAD lexicon + structural)
  - affect_arousal     (NRC VAD lexicon + structural)
  Behavioral (7):
  - behavioral_frustration, _boredom, _curiosity,
    _anxiety, _joy, _warmth, _arousal
    |
    v  [classifier.py: classify_from_output]
observed_type (infrastructure/philosophical/companion/consolidation/exploratory)
    |
    v  [feelings.py: classify_feeling]
discrete feeling label + affordances (frustrated/curious/bored/etc.)
    |
    v  [baselines.py: compute_deviations]
z-scores against per-type Welford baselines
    |
    v  [baselines.py: format_prediction_error]
injection block (only if type mismatch or signal deviation)
    |
    v  injected into next heartbeat pulse prompt
```

*State persisted in `agent/interoception/state.json` (version 5). Baselines self-calibrate after ~20 pulses per type.*
*Affect lexicon: `vad_lexicon.tsv` (NRC VAD v2, 44k entries, valence + arousal only).*
*Decay: 0.85 per pulse on all signals.*

#### Memory Companion Pipeline Detail (PER-8)

```
pulse prompt submitted to Agent SDK
    |
    v  [UserPromptSubmit hook fires]
    |
    v  [extractor.py: extract_topics]
topics dict:
  - instructions_text  ([HUMAN]'s instructions, if any -- highest priority)
  - feeling_queries    (from interoception state -- e.g. "exploration discovery")
  - categories         (infrastructure/identity/companion/etc. -- broad, high threshold)
    |
    v  [hook.py: BM25 queries against .claude-rag/]
    |   Direct import via importlib (no MCP, no sys.path)
    |   BM25-only (fast, no Ollama dependency)
    |
    v  [hook.py: threshold filter]
    |   Instructions/feelings: score >= 2.0
    |   Categories: score >= 5.0 (must be strong match)
    |
    v  [state.py: decay suppression]
    |   Skip queries offered recently (weight > 0.4, decay 0.85/pulse)
    |
    v  additionalContext injection (~50-60 words)
        Pointers only -- file paths + query hints
        Model decides whether to follow up via RAG MCP tool
```

*State persisted in `agent/memory_companion/companion_state.json`.*
*Design principle: pointers, not content. The model's intelligence decides relevance, not heuristics.*

## Growth Conventions

*Where new things go. The "where does this belong?" question answered before it causes paralysis.*

| What you're creating | Where it goes | Example |
|---------------------|---------------|---------|
| Generated artifacts (reports, logs, traces) | `output/` under a named subdir | `output/heartbeat_reports/`, `output/texture_traces/` |
| Notes, observations, pulse reflections | `files/notes/YYYY-MM/` | `files/notes/2026-02/2026-02-21_the-signal.md` |
| Consolidation proposals | `output/staging/consolidation/` | `output/staging/consolidation/proposal_2026-02-21.md` |
| Finished consolidation work | `output/consolidated/` | `output/consolidated/2026-01-07_cross-model-dialogues.md` |
| New tool or subsystem | Root directory (same level as `saliency-detector/`) | `new-tool/` |
| Artifacts for another instance to pick up | `drops/inbox/` | `drops/inbox/reading-list.md` |
| Dormant or superseded project dirs | `archive/` | `archive/tools/`, `archive/TRAJECTORY.md` |
| Test files | `tests/` in the matching tier | `tests/tier1_pure_logic/test_new_thing.py` |

**The principle:** Source and active systems live at root. Generated output goes in `output/`. Dead things go in `archive/`. Notes go in `files/notes/`. If you're not sure, `drops/inbox/` is the universal staging area — it gets checked.

**For the agentic loop:** If you create something and aren't sure where it belongs, put it in `drops/inbox/` with a note about what it is. It'll get triaged. Don't create new root directories without good reason — the root should stay under 25 items.

## Mode-Specific Notes

### CLI Mode (Claude Code)
- Entry: `claude` command in this directory
- Reads: `CLAUDE.md` for instructions
- Has: Full tool access, RAG via MCP, Linear via MCP

### Heartbeat Mode
- Entry: `python agent/heartbeat.py` (manual only, not scheduled)
- Runner: `agent/runner.py` via Agent SDK (Claude CLI subscription)
- Hooks: memory companion (UserPromptSubmit), safety (PreToolUse), sub-agent logging (PostToolUse)
- MCP servers: RAG (stdio), companions (stdio), Linear (HTTP)
- Outputs: sessions, run_archives, companion_logs
